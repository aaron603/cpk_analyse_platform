一、项目背景
--------------------------------------------------------------------------------
Zillnk 公司设计的产品需委托合作 EMS 厂商（深科技、立讯）加工生产。生产过程中，
各测试工站的测试数据会上传至工厂 MES 系统；与此同时，测试站本地也会保存 xlsx 和
json 两种格式的本地测试记录。

本平台旨在为 Zillnk 质量工程师提供三大数据分析功能，辅助产品测试质量管控与 CPK
分析工作。

二、测试工序类型说明
--------------------------------------------------------------------------------
| 工序标识        | 中文名称             | 说明                              |
|----------------|---------------------|----------------------------------|
| Board Test      | 单板测试             | PCB 单板功能性验证                  |
| FT1             | Calibration 校准     | 模块校准测试                        |
| Aging           | 老化测试             | 高温/长时间老化测试                  |
| FT2             | Performance 性能     | 整机性能指标测试                    |
| Robustness      | 温循测试             | 温度循环可靠性测试                   |

三、本地文件存储结构说明
--------------------------------------------------------------------------------
用户从各测试工站拷贝数据时，将每台工站的数据单独存放，并按"工站类型+编号"命名顶层
文件夹（如 FT1 1、FT1 2、FT1 3 …）。同类工站可能存在多台设备，同一模块在测试失败后
可能被安排到另一台同类工站重新测试，因此同一条码的测试记录可能分散在同类工站的多台
设备数据中。

【规范结构（新版）】
[工站文件夹]/                  ← 用户命名，如 "FT1 1"（工站类型+编号）
  TestResult/                  ← 站内固定根目录名称（固定）
    [产品类别]/                 ← 按产品型号分类，如 N3、N40（固定一级）
      [工站类型]/               ← 按工站功能分类，如 FT1、FT2、VSWR（固定一级）
        [可变中间层级]/          ← 0~N 层，不同工程师设置可能有差异（程序自动跳过）
        debug/                  ← 调试测试记录（自动跳过，不参与 CPK 分析）
        [产品条码] 或 [条码1_条码2]/  ← 产线正式测试目录
                               ← 单模块测试：以单个条码命名
                               ← 双模块同测：以两个条码联合命名，如 BC1_BC2
          [年月日时分秒]/        ← 每次测试生成一个时间戳命名的文件夹
            Test_Result_*_[条码].xlsx    ← 每个条码一个xlsx（含条码名）
            [条码]_*_MEASUREMENT_*.json ← 每个条码一个json（含条码名）
            ate_test_log.html   ← 人工可读测试报告（后续 AI 故障分析）
            ate_test_log.log    ← 详细日志（后续 AI 故障分析）
            [测试子项目]/        ← 各子项截图/结果（后续 AI 分析用，自动跳过）
            file_bk/            ← 环境备份（非测试数据，自动跳过）
          [年月日时分秒]/        ← 同一条码的第二次测试（可能 pass 或 fail）
            ...

【兼容结构（历史数据）】
[工站文件夹]/
  [分组子目录]/                ← 可选，如 X11、X11_X11（单/双板分组）
    [条码] 或 [条码1_条码2]/   ← 条码文件夹（支持两模块同测）
      [年月日时分秒]/
        Test_Result_*.xlsx
        *_MEASUREMENT_*.json
        ...

注意事项：
  - debug/ 目录下的测试记录自动跳过（仅统计产线正式测试数据）
  - file_bk/、RU1_Log_*、TM1_Log 等辅助目录自动跳过
  - 同一模块在多台同类工站上的测试记录会被汇总，选取最新一次成功记录
  - 双模块同测（条码1_条码2文件夹）：程序为每个条码独立匹配其xlsx/json文件，
    成功判定也独立进行，互不影响
  - 非数值型测试子项（如版本号、状态字符串）和固定值子项（方差=0）
    不参与 CPK 计算，不在 HTML 报告中显示

四、测试记录 xlsx 文件格式说明
--------------------------------------------------------------------------------
每个 xlsx 文件包含多个 Sheet 页，每个 Sheet 对应一个测试大项目。

各 Sheet 列定义：
| 列名         | 数据类型 | 说明                             |
|-------------|---------|----------------------------------|
| point_name  | string  | 测试子项目名称（CPK 分析的基本单元）  |
| result      | string  | 测试结果：pass（成功）/ fail（失败）  |
| data        | number  | 测试实测值                         |
| limit_low   | number  | 测试下限（LSL）                    |
| limit_high  | number  | 测试上限（USL）                    |
| start_time  | datetime| 该子项测试开始时间                  |
| stop_time   | datetime| 该子项测试结束时间                  |

【功能模块一】测试站本地测试数据分析（当前版本实现）

### 5.1 数据提取流程

输入：
  - 用户提供的发货 Excel 文件，必须包含 "PrdSN" 列（模块条码列）
  - 用户配置的各类测试工站数据文件夹（支持多种工站类型，每类可配置一个或多个根目录）
  - 每类工站需用户自定义类型标签（如 FT1、FT2、Aging 等）

提取规则：
  1. 读取发货 Excel 的 PrdSN 列，得到待分析条码列表
  2. 对每类工站，遍历用户指定的所有该类型工站数据根目录
     【需求 #1】同一类型工站可能有多台设备（如 FT1 1 / FT1 2 / FT1 3 ...），
     同一模块可能在多台同类站上都有测试记录；用户可在 GUI 中为同一工站
     类型配置多个文件夹。程序对所有同类型文件夹统一遍历，汇总该条码的
     全部测试记录后再进行后续筛选。
  3. 对每个条码，找到该条码对应的所有测试记录文件夹
     - 条码匹配：条码文件夹名包含该条码即视为匹配
       · 单模块测试：文件夹名即为单个条码，如 WV261331368/
       · 双模块同测：文件夹名含两个条码，如 WV261330001_WV261330011/
         程序分别为每个条码匹配其对应的 xlsx/json 文件（文件名中含条码）
     - 可跨越任意深度的子目录层级（条码文件夹直接下层为时间戳文件夹）
     【需求 #4】遍历时自动跳过名为 "debug"（不区分大小写）的子目录，
     以及 file_bk、RU1_Log_*、TM1_Log 等辅助目录，确保 CPK 分析
     仅统计产线正式测试数据，不含调试测试记录，同时提升遍历效率。
  4. 对该条码跨所有同类站的全部测试记录，过滤出"测试成功"的记录
     - 测试成功判定：该条码对应 xlsx 文件的所有 Sheet、所有测试子项的
       result 均为 pass（非 fail），即该模块本次测试全部通过
     - 双模块同测时，两个条码的成功与否相互独立，分别判断
     - 若仅有 json 文件（无 xlsx），以 DutInfo.Result 字段判断是否通过
  5. 从成功记录中选取"最新一次"
     - 最新判定：以 xlsx 中 start_time 列的最小值作为时间基准进行比较
     - 若无 xlsx（json-only），以 DutInfo.StartTime 作为时间基准
  6. 将选定的 xlsx 和 json 文件复制到用户指定的输出目录
     - 自动创建两个子目录：xlsx/ 和 json/
     - 目标文件名保持原文件名不变（原文件名中已含条码和时间戳，天然唯一）
     - 不添加任何前缀，确保 json 文件名符合客户要求的命名规范

输出：
  - 输出目录下，按工站类型分子目录存放提取出的 xlsx 和 json 文件
  - 【需求 #2】同时在输出目录下生成缺失条码汇总 Excel（见 5.5 节）

### 5.2 CPK 分析

分析对象：输出 xlsx 文件夹中的所有 xlsx 文件

条码提取规则（从 xlsx 文件名中提取）：
  文件名格式：Test_Result_<YYYYMMDDHHMMSS>_<条码>.xlsx
  提取方式：优先用正则匹配 14 位时间戳后的部分作为条码；
            若匹配失败，取最后一个 "_" 之后的部分作为回退。
  提取出的条码与测试值关联存储，用于 HTML 报告条码查询功能。

分析规则：
  1. 以 Sheet 为单位，以 point_name 为子项进行 CPK 分析
  2. 上下限统一规则：以时间最新的文件中对应子项的 limit_low/limit_high 作为统一门限
     （因测试过程中门限可能被调整）
  3. 跳过规则（以下子项不参与 CPK 计算，不在报告中显示）：
     a. 测试值为非数值型（如版本号 "3.31.11"、状态字符串 "P1A" 等）
     b. 所有测试值完全相同（固定值，方差为 0，如恒定配置参数）
     c. 样本量不足（有效数值样本 < 2）
  4. Sheet 级过滤：若某 Sheet 内所有子项均被跳过，该 Sheet 标签页在 HTML 报告中隐藏
     （不显示空表格）

CPK 计算公式：
  - σ（标准差）= 样本标准差（ddof=1）
  - Cp  = (USL - LSL) / (6σ)
  - Cpl = (均值 - LSL) / (3σ)
  - Cpu = (USL - 均值) / (3σ)
  - Cpk = min(Cpl, Cpu)
  - 当 LSL 或 USL 缺失时，相应指标不计算

### 5.3 HTML 报告展示规格

报告以独立 HTML 文件方式输出，支持浏览器本地打开，无需网络。

报告整体结构：
  - 顶部：平台标题 + 工站类型选项卡（Tab）
    标题副行格式：生成时间 | 共 N 种工站类型：FT1 × 3台，VSWR × 1台
    （N = 不同工站类型数；台数 = 该类型配置的文件夹数量）
  - 每个工站 Tab 内：包含该工站的 Sheet 选项卡
  - 每个 Sheet Tab 内：依次为搜索区、数据汇总表、正态分布图

【背景说明：CPK 数据对客户隐藏的设计意图】
  HTML 报告有时直接提供给客户查看。当 CPK 数据不理想时，不希望客户在打开页面时
  立即看到不良的 CPK 数值。因此，所有 CPK 相关数据默认完全隐藏：
    - 表格中的 Cp / Cpl / Cpu / Cpk 列隐藏
    - 正态分布图下方信息栏中的 Cpk 数值隐藏
  搜索框外观上是普通的"搜索测试项目"过滤框，不透露任何 CPK 相关提示。
  Zillnk 内部用户在搜索框中键入 "cpk"（不区分大小写）后，所有隐藏内容才同步显示；
  清除搜索内容后，CPK 数据重新隐藏。

【区域一：搜索与过滤栏】
  - 文本搜索框：占位符文字为"搜索测试项目..."，不暗示 CPK 功能的存在
  - 名称过滤：输入关键字实时过滤表格行（"cpk" 关键字被剔除，不影响行过滤）
  - CPK 隐藏开关（内部功能）：当搜索框内容包含 "cpk" 时，
      · 汇总表中的 Cp / Cpl / Cpu / Cpk 列同步显示
      · 正态分布图下方信息栏中的 Cpk 数值同步显示（含颜色标注）
    搜索框清空或删除 "cpk" 字样后，上述内容自动重新隐藏
  - 数值范围搜索：选择测试子项 + 输入数值范围，查询命中条码列表
    · 最小值 < 最大值：范围搜索，结果标题显示"范围搜索: [lo ~ hi]"
    · 最小值 = 最大值：精确匹配（浮点安全容差），结果按值排序，
      标题显示"精确匹配: point = value"；可用于定位特定测量值的模块

【区域二：测试数据汇总表】
  默认显示列（对客户可见）：
  | 测试项目(point_name) | 样本数 | 均值 | 标准差 | 最小值 | 最大值 | 下限(LSL) | 上限(USL) |

  CPK 列（默认隐藏，仅搜索框输入 "cpk" 后展示）：
  | Cp | Cpl | Cpu | Cpk |
  颜色标注规则（仅在显示时生效）：Cpk < 1.0 红色，1.0–1.33 橙色，≥ 1.33 绿色

  交互：
  - 点击某行 → 下方正态分布图切换为该测试子项

【区域三：正态分布图】
  - 默认显示第一个测试子项的正态分布图
  - 图表内容：
    - 正态分布密度曲线（基于均值和标准差）
    - 实测数据直方图（叠加显示）
    - 均值线（绿色虚线，标注 μ）
    - LSL 下限线 / USL 上限线（红色虚线，标注数值）
      【注】无论数据是否落在限值范围内，LSL/USL 线始终绘制；
      X 轴自动扩展以包含 LSL 和 USL（各加 0.4σ 边距），确保限值可见
  - 图表下方信息栏：默认显示 测试项目名、N、μ、σ；
    Cpk 数值默认隐藏，搜索 "cpk" 后同步显示（含颜色标注）
  - 用户可通过点击表格行切换显示的测试子项

### 5.5 缺失条码汇总报表【需求 #2】

分析运行结束后，自动在输出目录下生成 Excel 文件 `missing_barcodes.xlsx`。
当所有条码均已找到时，仍生成文件，但正文为空（只有表头）。

文件格式：
  - 每个工站类型一个 Sheet 页（Sheet 名 = 工站类型标签，如 FT1、VSWR）
  - 各 Sheet 列定义：

  | 列名               | 说明                                          |
  |--------------------|----------------------------------------------|
  | PrdSN              | 模块条码                                      |
  | 状态               | 未找到测试记录 / 无通过测试记录 / 有通过但无xlsx  |
  | 总测试次数         | 在所有同类站文件夹中找到的测试记录总数            |
  | 通过次数           | 其中测试全部通过（all-pass）的记录数             |
  | 最新测试时间       | 最新一次测试（不限是否通过）的时间戳              |
  | 备注               | 附加说明（如：仅 json 无 xlsx；文件夹访问错误等） |

  - 各 Sheet 底部附加一行汇总：总条码数 / 已找到 / 缺失数
  - 若某工站无任何缺失条码，对应 Sheet 仅保留表头和汇总行

### 5.6 运行日志文件【需求 #v1.4】

每次点击"开始分析"，自动在输出目录下生成带时间戳的日志文件：
  analysis_log_YYYYMMDD_HHMMSS.txt

  - 内容与 GUI 日志窗口完全同步（逐行 flush 写入）
  - 覆盖所有阶段：条码读取、工站遍历提取、CPK 分析、报告生成
  - 分析结束（完成/出错/中止）后文件自动关闭
  - 用途：便于问题复现与远程排查，可直接分析定位原因

【需求 #3】GUI 日志详细程度要求：
  日志应涵盖以下所有过程事件，确保问题可追溯：

  启动阶段：
  - 条码文件读取结果（文件路径、总条码数、去重后条码数）
  - 各工站类型的文件夹数量和具体路径列表
  - 输出目录路径确认

  逐条码处理（数据提取阶段）：
  - [OK]  成功提取：条码 + 所在文件夹 + 测试时间 + 是否有 json
  - [WARN] 找到测试记录但全部失败：条码 + 总测试次数 + 最近失败时间
  - [WARN] 找到测试记录，pass 但无 xlsx 可供 CPK 分析：条码 + 说明
  - [WARN] 完全未找到：条码（需在缺失报表中记录）
  - [ERROR] 文件读取异常：条码 + 文件路径 + 错误原因

  工站汇总（每个工站类型完成后）：
  - 成功提取数 / 无通过记录数 / 未找到数 / 错误数 / 总数

  CPK 分析阶段：
  - 每个工站开始分析时：xlsx 文件夹路径 + 文件数量
  - 跳过的固定值子项：point_name + 原因（方差为0）
  - 分析完成：sheet 数 + point_name 总数
  - 异常文件：文件名 + 错误原因

  输出阶段：
  - 缺失条码报表保存路径 + 各工站缺失数量统计
  - HTML 报告保存路径 + 文件大小
  - 运行日志文件路径
  - 总耗时

【需求 #v1.4-1】停止分析
  "开始分析"按键在运行期间变为红色"停止分析"；点击后设置取消标志，
  程序在下一个条码边界（数据提取阶段）或下一个工站边界（CPK阶段）处
  安全退出，不会强制中断正在进行的文件读取操作。

【需求 #v1.4-2】配置持久化
  程序启动时自动从同目录 app_config.json 加载上次配置：
  发货 Excel 路径、输出目录、工站类型及文件夹列表。
  配置保存时机：点击"开始分析"时 + 正常关闭窗口时。
  文件不存在（首次运行）时静默忽略，保留默认工站行。

【需求 #v1.4-3】工站配置键盘导航
  在工站类型输入框或路径输入框内：
  - 按 ↑ 移动焦点到上一行同列输入框
  - 按 ↓ 移动焦点到下一行同列输入框
  边界行（首行按↑、末行按↓）不响应，不循环。

七、技术选型
================================================================================
  - GUI 框架：Python tkinter + ttk（跨平台，无需额外安装）
